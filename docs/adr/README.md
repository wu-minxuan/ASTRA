# 架构决策记录

本目录用于记录 ASTRA 项目中的重要架构决策。

ADR 是 Architecture Decision Record 的缩写，即架构决策记录。它用于解释项目为什么选择某个方向，而不是只记录最终结果。

## 什么时候需要 ADR

以下情况应考虑新增 ADR：

- 选择或更换关键技术栈
- 确定长期架构方向
- 改变模块边界
- 引入新的数据源、框架、服务或运行时
- 调整测试策略、验证策略或部署策略
- 做出会影响多个阶段的取舍

## 文件命名

ADR 文件使用递增编号：

```text
0001-title-in-kebab-case.md
0002-title-in-kebab-case.md
```

标题使用英文路径名，正文使用中文。

## ADR 模板

```md
# ADR 0001：决策标题

## 状态

提议中 / 已接受 / 已废弃 / 已替代

## 背景

说明为什么需要做这个决策。

## 决策

说明最终选择。

## 理由

说明选择该方案的主要原因。

## 影响

说明这个决策会带来的影响、限制和后续注意事项。

## 替代方案

说明曾考虑过但未采用的方案。

## 重新评估条件

说明什么情况下需要重新审视这个决策。
```
