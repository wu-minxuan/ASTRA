# 原子任务规格模板

本模板用于定义 ASTRA 项目的原子任务。进入执行模式前，任务应具备清晰边界、验收标准和验证方式。

小任务可以先在对话中形成临时任务规格；中大型任务应写入 `docs/tasks/`。

## 任务标题

`PX-TYY：任务名称`

## 背景

说明为什么需要这个任务，相关上下文是什么。

## 目标

说明本任务要完成的单一目标。

## 非目标

说明本任务明确不做什么，避免任务范围扩张。

## 允许修改范围

列出本任务允许创建、修改或删除的文件和目录。

## 禁止修改范围

列出本任务不得触碰的文件、目录、配置或行为。

## 输入

列出任务需要依赖的输入，包括文档、数据、配置、接口或用户确认信息。

## 输出

列出任务完成后应产生的结果，包括文件、接口、命令、报告或行为变化。

## 验收标准

用可判断的方式描述任务完成条件。

## 验证方式

列出必须运行的验证命令或人工检查项。

ASTRA 的验证类型固定为：

- 静态检查
- 单元测试
- 集成测试
- 端到端测试

## 风险点

说明可能的风险、阻塞条件或需要用户确认的事项。

## 收尾要求

说明任务完成后需要如何更新任务记录、文档或后续计划。
