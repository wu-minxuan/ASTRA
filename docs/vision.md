# ASTRA 项目愿景

ASTRA 是一个自适应策略研究与测试 Agent 系统，面向股票与多资产市场，支持从主题发现、股票池构建、基本面分析、研报分析，到量化因子研究、策略回测、模拟盘、策略 A/B 实验和持续迭代的研究闭环。

项目第一阶段聚焦 A 股市场，后续可扩展到港股、美股、ETF 和其他资产市场。

## 项目动机

ASTRA 同时服务于两个目标。

第一，满足个人业余投资研究需求。它应帮助用户更系统地发现主题、构建股票池、整理证据、分析风险，并逐步把投资想法转化为可跟踪、可复盘的研究过程。

第二，作为 Agent 工程实战项目。它将用于实践 AI Coding、Harness Engineering、Agent 工作流设计，以及后续模型部署和推理优化。

ASTRA 不追求一次性构建完整系统，而是通过阶段性 MVP，把复杂能力拆成可验证、可复用、可持续迭代的部分。

## 长期能力范围

ASTRA 长期希望支持以下能力：

- 主题发现与热点跟踪
- 股票池构建与更新
- 基本面数据分析
- 财报、公告、研报和新闻分析
- 产业链与概念关系梳理
- 量化因子研究
- 策略构建与回测
- 模拟盘跟踪
- 策略 A/B 实验
- 研究过程记录与结果复盘
- 多市场、多资产扩展

这些能力应逐步形成完整研究闭环，而不是孤立工具集合。

## 阶段推进原则

ASTRA 采用阶段化建设方式。每个阶段都应有明确的目标、范围和验收标准。

阶段性 MVP 是当前交付目标，不等于 ASTRA 的完整定义。阶段目标可以选择长期愿景中的一小段闭环，先做深、做稳、做可复用。

## 核心原则

ASTRA 的设计和使用应遵守以下原则：

- 可追溯：重要结论应尽量保留来源、时间、证据和推理过程。
- 可复现：重要研究流程应尽量能在相同输入和数据条件下复现。
- 可迭代：系统能力应支持从小闭环开始，逐步扩展和重构。
- 成本可控：大模型应优先用于高价值判断环节，而不是无差别处理全量数据。
- 人机协作：Agent 应辅助研究和开发，不替代用户的最终判断。
- 经验沉淀：每次重要讨论、决策和实验都应尽量沉淀为后续可继承的资产。

## 金融研究边界

ASTRA 是投资研究辅助系统，不是确定性荐股系统。

系统输出应区分事实、推断和假设。涉及股票、行业、主题和策略的输出，应尽量说明证据、不确定性和风险。

ASTRA 不应在缺少证据链的情况下给出确定性投资建议，也不应忽略数据时间点、未来函数、交易成本、停牌、涨跌停、退市等金融研究风险。

## 非目标

在项目早期，ASTRA 不追求：

- 直接接入真实交易并自动下单
- 覆盖所有市场和所有资产
- 构建无法验证的大而全 Agent
- 用大模型替代所有数据处理和量化分析
- 输出没有证据链的确定性结论
- 为未来可能需求提前设计复杂架构

## 成功标准

ASTRA 的长期成功不以代码规模衡量，而以研究闭环质量衡量。

一个成功的 ASTRA 应该能够：

- 把模糊研究问题转化为结构化研究流程
- 生成有证据、有边界、有风险提示的研究结果
- 记录每次研究和实验的输入、过程和输出
- 支持策略从想法到验证、跟踪和复盘的持续迭代
- 让用户在研究和开发过程中逐步积累自己的方法论
